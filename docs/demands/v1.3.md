# DistributionBox — 需求文档（v1.3）

> v1.3 主题：建立“系统可用性自动化测试（模拟网页点击）”基线能力，并输出问题清单供业务筛选。

- 版本：v1.3
- 状态：进行中
- 更新时间：2026-02-11
- 关联系统：配电箱管理系统（`~/project/distributionbox`）
- 关联规则：发现一个可复现问题，新增后续版本文档（v1.x）记录

---

## 1. 背景与目标
用户希望对系统进行可用性检测，通过模拟真实用户点击网页执行关键流程，验证系统是否可正常使用。允许在测试中使用模拟数据。

目标：
1. 建立可重复执行的 UI 自动化测试（优先 E2E/Smoke）。
2. 覆盖核心业务路径（登录、导航、关键页面访问与基础操作）。
3. 发现问题后，先文档化记录，再由用户决定是否进入修复。

---

## 2. 范围
### 2.1 本次测试范围
- 登录流程可用性
- 侧边栏导航可用性
- 关键页面打开与基础数据加载
- 退出登录与未授权重定向行为

### 2.2 数据策略
- 优先使用现有环境数据
- 必要时允许注入模拟数据（UI 或数据库）
- 数据注入动作必须可追溯、可回滚（至少可人工清理）

### 2.3 暂不包含
- 性能压测
- 安全渗透测试
- 大规模并发稳定性

---

## 3. 执行要求（给 Codex/Claude）
1. 在不破坏现有业务代码的前提下，新增测试代码与执行脚本。
2. 采用“最小侵入”策略，不对主业务逻辑做无关改动。
3. 输出测试报告：通过项、失败项、失败复现步骤、日志/截图位置。
4. 如发现 bug：先记录到需求文档，不直接扩大修复范围。

---

## 4. 期望输出结果（验收）
1. 产出可执行的系统可用性测试脚本（可重复运行）。
2. 产出一次完整测试报告（含通过/失败统计）。
3. 每个可复现问题形成独立需求版本（v1.x）候选条目，供用户筛选价值后再排期修复。
4. 所有发现的问题均先写文档，不擅自做超范围改动。
