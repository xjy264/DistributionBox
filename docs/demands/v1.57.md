# v1.57 需求文档

## 问题
维保工作记录保存时后端报：
- `Request method 'POST' is not supported`（update 接口在旧后端不存在）
- `LocalDateTime` 解析失败（`2026-02-15 10:00:00` 不能按默认格式反序列化）

## 目标
1. 保持 update->save 的 405 兼容回退。
2. 保存请求中的 `inspectionTime` 统一转换为后端可解析格式（ISO：`yyyy-MM-ddTHH:mm:ss`）。

## 验收标准
- 点击保存不再出现 LocalDateTime 解析报错。
- 新旧后端实例下都能保存成功并回显。
