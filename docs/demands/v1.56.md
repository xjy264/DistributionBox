# v1.56 需求文档

## 问题
维保工单详情页“修改维保工作记录表”保存时报错 405（Method Not Allowed）。

## 目标
1. 在不影响现有新后端 `/maintenance-task/update` 的前提下，兼容仍在运行旧后端实例的保存链路。
2. 用户点击保存必须有明确成功/失败反馈，不允许“没反应”。

## 验收标准
1. 当 `/maintenance-task/update` 返回 405 时，前端自动回退调用 `/maintenance-task/save`。
2. 回退保存成功后，弹窗关闭并刷新回显。
3. 非 405 错误时，前端展示后端错误信息或通用失败提示。
