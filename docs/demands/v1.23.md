# v1.23 需求文档（管理员全权限 + users/roles/menus 限权）

## 目标
在现有登录体系基础上完成权限收口：管理员保留全量访问能力，普通用户禁止访问 users/roles/menus 相关能力，并在前后端双重拦截，避免越权。

## 需求范围
1. 后端权限控制
   - 对以下资源实施管理员专属访问：
     - `/users/**`、`/roles/**`、`/menus/**`
     - 同时兼容当前项目实际接口前缀（`/user/**`、`/role/**`、`/menu/**`）
   - 非管理员访问上述资源时返回 403（Forbidden），不得出现 500。
   - 其他业务接口（如配电箱、巡检、检测、维保等）保持管理员可访问。

2. 前端权限控制
   - 普通用户菜单中隐藏 users/roles/menus 入口。
   - 即使手工输入 `/users`、`/roles`、`/menus` 路径，也要通过路由守卫拦截并跳转到 `/box`。

3. 稳定性要求
   - 登录流程保持可用。
   - 现有主要业务页面可正常访问与构建。

## 验收标准
1. 后端编译通过：`cd backend && mvn -DskipTests compile`
2. 前端构建通过：`cd frontend && npm run build`
3. 角色验证：
   - Admin：可访问 users/roles/menus + 其他业务页面。
   - User：users/roles/menus 被后端 403 拒绝，前端菜单隐藏且直达路由被拦截跳回。
