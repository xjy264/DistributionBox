# v1.27 需求文档

## 需求背景
当前“配电箱详情”中图片上传存在工程问题：
1. 选择图片后页面无法稳定回显；
2. 图片信息未稳定落库（字段未更新或值无效）；
3. 影响详情页二次打开后的图片可见性。

## 目标范围
仅本次先修复“配电箱详情”模块，覆盖全部图片相关字段：
- `systemUrl`
- `firstUrl`
- `secondUrl`
- `thirdUrl`
- `fourthUrl`

## 预期结果
1. 在配电箱详情页选择图片后可立即回显；
2. 保存后数据库对应字段正确持久化；
3. 重新进入详情页，图片仍可正确显示；
4. 前后端统一使用工程化上传链路，避免临时拼接和不兼容返回结构。

## 实施原则
1. 上传存储格式采用“数据库存相对路径/URL，文件落磁盘（或静态资源目录）”；
2. 后端提供清晰的上传接口与返回结构；
3. 前端上传组件统一走上传接口，拿到路径后回填字段；
4. 兼容旧数据和空值显示，避免页面报错。

## 验收标准
1. 任意图片字段可上传、回显、保存、重开可见；
2. 数据库字段可查到有效路径值；
3. 构建通过，核心流程可复现通过。
