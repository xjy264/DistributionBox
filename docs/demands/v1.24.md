# v1.24 需求文档（修复注册接口 + 强制后端管理员权限）

## 背景
当前存在两项问题：
1. 前端注册页调用了错误接口（`/auth/register`），导致注册 404，UI 无法完成注册流程。
2. 后端权限约束未严格生效，普通用户仍可访问用户/角色/菜单相关接口，存在越权风险。

## 目标
1. 前端注册页面改为调用后端真实注册接口，确保 UI 可成功注册。
2. 后端强制管理员权限：普通用户访问以下路径必须返回 403，管理员可访问：
   - `/user/**`、`/users/**`
   - `/role/**`、`/roles/**`
   - `/menu/**`、`/menus/**`

## 具体要求

### 后端
- 检查 `SecurityConfig` 受限路径匹配（重点排查 `/users` vs `/user` 等单复数不一致）。
- 对上述单复数路径全部限制为 `ADMIN`。
- 无权限时通过 `accessDeniedHandler` 返回 403。

### 前端
- 检查 `Register.vue` / 相关 API，修正注册接口路径，不再请求 `/auth/register`。
- 注册成功后给出用户可见反馈，并可返回登录页。

## 复测清单（必须）
1. 管理员登录可用；管理员访问 users/roles/menus 页面可用。
2. UI 注册普通用户成功；普通用户登录成功。
3. 普通用户前端菜单不显示 users/roles/menus；手输路由被拦截到 `/box`。
4. 普通用户直连后端：
   - `GET /user/page?pageNum=1&pageSize=1` -> 403
   - `GET /role/page?pageNum=1&pageSize=1` -> 403
   - `GET /menu` -> 403

## 构建要求
- backend: `cd backend && mvn -DskipTests compile`
- frontend: `cd frontend && npm run build`
