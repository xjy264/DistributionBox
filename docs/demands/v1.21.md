# v1.21 需求文档（Phase 2：抢修同构 task-item）

## 目标
将“抢修”从旧的 `记录 + 关联表` 模式，按 v1.20 巡检 Phase1 同构改为：

- 抢修主单：`sys_emergency_repair_task`
- 抢修子单：`sys_emergency_repair_item`

实现“一次抢修任务可覆盖多个配电箱；每箱在该任务下仅一条明细”。

## 范围
1. 新增数据库迁移：创建抢修 task/item 两张表，包含 `UNIQUE(task_id, box_id)` 与必要外键。
2. 后端新增 repair-task / repair-item 全套分层（entity、mapper、service、controller、DTO）。
3. 新接口：
   - `POST /repair-task/save`
   - `GET /repair-task/page`
   - `GET /repair-task/{id}`
   - `DELETE /repair-task/{id}`
   - `GET /repair-item/page?boxId=`
4. 前端：
   - 抢修页面（Fix.vue）切换到新 task 接口（最小可用）
   - 配电箱详情页“抢修记录”改为按箱明细历史（repair-item/page）
5. 兼容：旧 `/emergency-repair/**` 接口保留，不删除。

## 验收
- 可新增/编辑/删除抢修任务（主单+items）。
- 任务列表按主单展示；配电箱详情按明细历史展示。
- 后端编译与前端构建通过。
