# v1.55 需求文档

## 问题
维保工单详情页中“维保工作记录表”点击“修改”后，保存无明显反馈，用户判断为接口问题。

## 目标
1. 详情页“保存”应调用维保工单更新接口（非新建接口）。
2. 更新成功/失败均给出前端提示。
3. 后端更新接口可接收并持久化维保工作记录表字段，至少包含：
   - `maintenanceType`
   - `maintenanceLocation`
   - `inspectionTime`
   - `maintenanceContent`
   - `maintenanceSummary`
   - `taskNo`

## 验收标准
1. 在维保工单详情页打开“修改维保工作记录”弹窗，点击“保存”后，前端实际请求更新接口 `/maintenance-task/update`。
2. 更新成功后弹窗关闭并提示“保存成功”，页面数据刷新为最新值。
3. 更新失败后页面不静默，显示错误提示。
4. 后端更新接口能成功更新上述字段并落库。
5. 本地至少完成前后端编译/构建验证通过。
