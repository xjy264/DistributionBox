# v1.20 需求文档（巡检/抢修重构为“主单 + 子单”）

## 一、目标
将当前“记录与配电箱多对多”模型，重构为：

- **主单（批次）**：表示“一次巡检 / 一次抢修任务”
- **子单（明细）**：表示“该批次下某一个配电箱的一条记录”

即：
- 一次任务可覆盖多个配电箱
- 每个配电箱在该次任务里有且仅有一条明细记录
- 页面与查询均以“主单可汇总、子单可追踪”为核心

---

## 二、业务语义（确认版）

### 2.1 巡检
- 一次巡检 = 1 个**巡检主单**
- 主单下 N 条**巡检明细**（每条对应 1 个配电箱）
- 配电箱历史巡检记录来自“明细表”

### 2.2 抢修
- 一次抢修 = 1 个**抢修主单**
- 主单下 N 条**抢修明细**（每条对应 1 个配电箱）
- 配电箱历史抢修记录来自“明细表”

---

## 三、数据库设计（建议）

> 说明：以下为新结构建议，不直接删除旧表，采用平滑迁移。

### 3.1 巡检主表 `sys_inspection_task`
字段建议：
- `id` PK
- `task_no` 任务单号（唯一）
- `inspection_user` 巡检人
- `guardian_user` 监护人
- `inspection_time` 巡检时间（任务级）
- `remark` 任务备注
- `created_at` / `updated_at`

### 3.2 巡检明细表 `sys_inspection_item`
字段建议：
- `id` PK
- `task_id` FK -> `sys_inspection_task.id`
- `box_id` FK -> `sys_box.id`
- 明细巡检字段（开关型号、电流、电压、A/B/C 起动运行、图片、备注等）
- `created_at` / `updated_at`

约束建议：
- `UNIQUE(task_id, box_id)`：同一任务下同一配电箱仅一条记录

### 3.3 抢修主表 `sys_emergency_repair_task`
字段建议：
- `id` PK
- `task_no` 任务单号（唯一）
- `report_user` 报修人
- `report_time` 报修时间
- `fix_user` 抢修负责人
- `fix_time` 完成时间
- `remark`
- `created_at` / `updated_at`

### 3.4 抢修明细表 `sys_emergency_repair_item`
字段建议：
- `id` PK
- `task_id` FK -> `sys_emergency_repair_task.id`
- `box_id` FK -> `sys_box.id`
- 明细字段（故障现象、原因、过程、更换元器件、图片、备注）
- `created_at` / `updated_at`

约束建议：
- `UNIQUE(task_id, box_id)`

## 四、接口改造（建议）

## 4.1 巡检
- `POST /inspection-task/save`
  - 入参：主单信息 + `items[]`（每项含 boxId 与该箱明细）
- `GET /inspection-task/page`
  - 主单分页（显示覆盖箱数、任务时间、负责人）
- `GET /inspection-task/{id}`
  - 主单详情 + 明细列表
- `GET /inspection-item/page?boxId=...`
  - 某配电箱的巡检明细历史

## 4.2 抢修
- `POST /repair-task/save`
- `GET /repair-task/page`
- `GET /repair-task/{id}`
- `GET /repair-item/page?boxId=...`

---

## 五、前端交互改造（建议）

### 5.1 新增任务（巡检/抢修）
- 第一步：创建主单（任务基础信息）
- 第二步：选择多个配电箱
- 第三步：为每个配电箱填写（或批量复制后微调）明细
- 保存后生成 1 个主单 + N 条子单

### 5.2 配电箱详情页
- 显示该箱“明细历史”而非“任务关系表”
- 可点击进入对应任务主单查看同批次其他配电箱情况

### 5.3 列表页
- 任务管理页：按主单展示（任务视角）
- 配电箱详情：按明细展示（设备视角）

---

## 六、迁移策略（平滑）

1. 新增表（不动旧表）
2. 新接口/新页面先接入新表
3. 编写一次性迁移脚本：
   - 旧 `sys_inspection + sys_inspection_box` -> 新 task/item
   - 旧 `sys_emergency_repair + sys_emergency_repair_box` -> 新 task/item
4. 验证通过后，旧接口降级为只读或下线

---

## 七、验收标准

1. 可创建“一个任务覆盖多个配电箱”的巡检/抢修主单
2. 同一任务中每个配电箱仅一条明细（唯一约束生效）
3. 配电箱详情能看到该箱所有历史明细
4. 任务详情可看到该任务下全部配电箱明细
5. 老数据可迁移并可查询

---

## 八、实施建议（分期）

- **Phase 1**：建表 + 后端新接口 + 最小可用前端（先巡检）
- **Phase 2**：抢修同构改造
- **Phase 3**：旧数据迁移 + 兼容清理
