# v1.22 需求文档（统一图片上传接口 + 全站图片字段上传改造）

## 目标
建立统一图片上传能力，替换项目内“手输图片 URL”的录入方式，统一为“本地上传 -> 自动回填 URL -> 可预览”，并确保历史 URL 数据兼容。

## 范围
1. 后端统一上传
   - 新增统一接口：`POST /files/image/upload`（multipart/form-data，字段名 `file`）。
   - 返回标准化结构：`url`、`filename`、`size`、`contentType`、`uuid` 等。
   - 复用可配置存储目录 `app.files.upload-path`，访问地址基于 `app.files.base-url`。
   - 安全校验：
     - 仅允许图片扩展名：jpg/jpeg/png/webp/gif/bmp
     - MIME 必须为 `image/*`
     - 文件大小限制（默认 10MB，可配置）
     - 拦截空扩展名/非法扩展名
   - 保留旧接口 `/files/upload` 兼容历史调用，但内部统一走新逻辑。

2. 前端全站图片录入改造
   - 统一使用 `ImageUpload.vue` 组件上传，不再手工输入图片 URL。
   - 上传成功后自动回填对应字段（保存的仍为 URL），并显示图片预览。
   - 覆盖页面：
     - 配电箱：`Box.vue`（系统图 + 图片1~4）
     - 巡检任务：`Inspection.vue`
     - 抢修任务：`Fix.vue`
     - 维保记录：`Maintenance.vue`
     - 配电箱详情弹窗内巡检/抢修表单：`BoxDetail.vue`
     - 用户头像：`User.vue`
     - 文件管理上传入口：`Files.vue`

3. 数据库与兼容
   - 新增迁移 SQL：扩展各业务表图片 URL 字段长度到 `VARCHAR(1000)`，兼容更长 URL。
   - 不改变既有字段语义，历史 URL 数据继续可读可用。

## 接口定义
- 路径：`POST /files/image/upload`
- 入参：`multipart/form-data`
  - `file`: 本地图片文件
- 出参（Result.data）：
  - `url`: 图片访问 URL（用于业务字段保存）
  - `filename`: 原始文件名
  - `size`: 字节大小
  - `contentType`: 文件 MIME
  - `extension`: 扩展名
  - `uuid`: 存储文件标识
  - `previewUrl`: 预览地址（`/files/preview/{uuid}`）

## 迁移策略
1. 先发布后端统一接口（兼容旧上传路径）。
2. 前端改造全部图片字段为上传组件。
3. 执行 `V27__expand_image_url_columns.sql` 扩容 URL 字段。
4. 历史数据不迁移：原有 URL 原样保留，表单可继续展示。
