---

## 变更说明

### 1) 数据库迁移
新增：`sql/migrations/V25__create_sys_inspection_task_item.sql`

- 新表 `sys_inspection_task`
  - `task_no` 唯一约束
  - 任务级字段：巡检人、监护人、巡检时间、任务备注
  - `created_at` / `updated_at`
- 新表 `sys_inspection_item`
  - `task_id` 外键到主表
  - `box_id` 外键到 `sys_box`
  - 明细字段（开关型号、电流电压、A/B/C 起动运行、图片、备注）
  - 关键约束：`UNIQUE(task_id, box_id)`

> 这样能保证：同一任务下，同一配电箱只能有一条明细。

### 2) 后端新增（按 task/item 分层）
新增实体：
- `InspectionTask`
- `InspectionItem`

新增 Mapper：
- `InspectionTaskMapper`
- `InspectionItemMapper`

新增 Service：
- `IInspectionTaskService` + `InspectionTaskServiceImpl`
- `IInspectionItemService` + `InspectionItemServiceImpl`

新增 Controller：
- `InspectionTaskController`
  - `POST /inspection-task/save`
  - `GET /inspection-task/page`
  - `GET /inspection-task/{id}`
  - `DELETE /inspection-task/{id}`
- `InspectionItemController`
  - `GET /inspection-item/page?boxId=...`

新增 DTO：
- `InspectionTaskSaveDto`
- `InspectionTaskViewDto`
- `InspectionItemViewDto`

实现细节：
- 保存任务时支持提交 `items[]`，后端按 `boxId` 去重后落库。
- 未传 `taskNo` 时自动生成（`ITyyyyMMddHHmmss`）。
- 旧接口 `/inspection/**` 保留，不删除，保证兼容。

### 3) 前端最小可用改造
修改页面：
- `frontend/src/views/Inspection.vue`
  - 列表改为任务视角（主单）
  - 新增/编辑改调用 `POST /inspection-task/save`
  - 任务详情读取 `GET /inspection-task/{id}`
  - 删除调用 `DELETE /inspection-task/{id}`
  - 允许选择多个配电箱，保存时为每个箱生成一条 item
- `frontend/src/views/BoxDetail.vue`
  - “巡检记录”改为“巡检明细历史”
  - 数据源改为 `GET /inspection-item/page?boxId=...`
  - 可看到该箱所属任务单号与明细信息

为什么这样做：
- 页面按“任务视角 + 设备视角”拆开，满足需求文档语义。
- 先做到可用且兼容，避免一次性破坏旧业务链路。

## 验证步骤（手工）
1. 执行数据库迁移 `V25`（或手动执行 SQL）。
2. 打开“巡检管理”页面，新建任务：
   - 选择多个配电箱（例如 A、B、C）
   - 填写巡检人、时间、明细
   - 保存
3. 在数据库验证：
   - `sys_inspection_task` 新增 1 条
   - `sys_inspection_item` 新增 3 条
4. 验证唯一约束：
   - 同一 `task_id` 下重复插入相同 `box_id` 应失败（`UNIQUE(task_id, box_id)` 生效）
5. 打开某个配电箱详情页：
   - 在“巡检明细历史”可看到该箱对应 item 历史
   - 可看到对应任务单号与巡检信息
6. 验证登录、配电箱列表与详情可正常访问。
