# v1.27 完成记录

## 本次改动
1. 修复配电箱详情页 5 个图片字段上传后不回显问题：`systemUrl/firstUrl/secondUrl/thirdUrl/fourthUrl`。
2. 修复图片保存后未稳定持久化问题：详情页新增“图片维护”编辑区，保存走 `/box/save`，字段可落库并在重进页面后可见。
3. 统一上传链路只保留一套：前端统一使用 `/files/image/upload`，后端统一由 `backend` 模块 `FilesController` 返回稳定结构。

## 根因
1. 前端 `ImageUpload` 仅兼容一种上传响应格式（`response.data.url`），后端/历史返回差异导致选图后 model 未更新。
2. 配电箱详情页此前只有图片文本展示，无稳定“上传-回填-保存”闭环。
3. 历史上存在两套实现思路，接口返回字段不统一，导致前端回显与落库链路偶发断裂。

## 具体修改文件
- `frontend/src/components/ImageUpload.vue`
  - 增强上传成功返回解析：兼容 `path/url/previewUrl`。
  - 统一预览地址解析：支持 `/files/{uuid}`、`/files/preview/{uuid}`、带 query 的旧值。
- `frontend/src/views/BoxDetail.vue`
  - 基础信息区将图片字段改为 `el-image` 可视回显。
  - 新增“图片维护”编辑区（5 个字段）+ 保存按钮，调用 `/box/save` 持久化。
  - 增加图片字段同步与空值兼容逻辑。
- `backend/src/main/java/com/distributionbox/controller/FilesController.java`
  - 上传返回统一结构：`path/url/accessUrl/previewUrl/uuid`。
  - 数据库存储统一相对路径：`/files/{uuid}`。
  - 预览接口 MIME 类型兜底，文件路径构造改为 `new File(uploadPath, uuid)`。
- `docs/demands/v1.27.md`
- `docs/done/v1.27.md`

## 接口与存储约定（统一后）
- 上传接口：`POST /files/image/upload`
- 成功返回：`Result{ code:200, data:{ path, url, previewUrl, uuid, ... } }`
- 数据库存储：`sys_box` 图片字段保存相对路径（示例 `/files/xxxxxx`）
- 页面预览：前端统一转为 `/api/files/preview/{uuid}` 进行展示

   - 点击“保存图片”后字段通过 `/box/save` 持久化；
