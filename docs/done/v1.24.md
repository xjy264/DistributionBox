# v1.24 完成记录（修复注册接口 + 后端管理员权限强制）

## 问题根因

1. **注册接口错误**
   - 前端 `Register.vue` 请求的是 `/auth/register`，而后端真实接口是 `POST /register`，导致注册 404。

2. **后端权限看似未生效的根因**
   - 运行中的服务里存在同名用户（如 `admin`）多条记录。
   - `UserDetailsServiceImpl#loadUserByUsername` 使用 `selectOne` 按用户名查用户，在同名数据场景会抛异常。
   - `JwtAuthenticationFilter` 捕获异常后静默忽略，最终认证上下文未建立，导致鉴权链路异常（表现为权限判定不稳定/管理员鉴权失败）。

## 修改点

### 1) 前端注册修复
- 文件：`frontend/src/views/Register.vue`
- 变更：
  - 注册接口从 `/auth/register` 改为 `/register`
  - 注册成功后增加可见提示：`ElMessage.success('注册成功，请登录')`
  - 成功后跳转登录页 `/login`

### 2) 后端鉴权稳定性修复 + 管理员权限生效
- 文件：`backend/src/main/java/com/distributionbox/security/UserDetailsServiceImpl.java`
- 变更：
  - 将 `selectOne` 改为 `selectList`
  - 同名用户时优先选择 `role` 非空记录，否则取第一条，避免 `selectOne` 多结果异常
  - 保证 JWT 过滤器可稳定建立认证上下文，`SecurityConfig` 中 ADMIN 路径约束可真正生效

### 3) 文档
- 新增需求文档：`docs/demands/v1.24.md`
- 新增完成文档：`docs/done/v1.24.md`

## 复测结果

> 复测环境：本地 `http://127.0.0.1:8888`

1. **管理员登录可用；管理员访问 users/roles/menus 对应后端接口可用**
   - `POST /login (admin/123456)`：200
   - `GET /user/page?pageNum=1&pageSize=1`：200
   - `GET /role/page?pageNum=1&pageSize=1`：200
   - `GET /menu`：200

2. **UI 注册普通用户成功；普通用户登录成功**
   - `POST /register` 新用户：200
   - `POST /login` 新用户：200（`role=TECHNICIAN`）

3. **普通用户前端菜单不显示 users/roles/menus；手输路由拦截到 /box**
   - 登录返回 `menus` 数量为 0（不含管理菜单）
   - 代码复核：
     - `frontend/src/layouts/MainLayout.vue`：普通用户过滤 `/users /roles /menus`
     - `frontend/src/router/index.ts`：普通用户手输上述路由时重定向 `/box`

4. **普通用户直连后端受限接口返回 403**
   - `GET /user/page?pageNum=1&pageSize=1`：403，`{"code":"403","msg":"无权限访问该资源"}`
   - `GET /role/page?pageNum=1&pageSize=1`：403，`{"code":"403","msg":"无权限访问该资源"}`
   - `GET /menu`：403，`{"code":"403","msg":"无权限访问该资源"}`
