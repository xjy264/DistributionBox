# v1.55 完成记录

## 根因
1. 维保工单详情页“修改维保工作记录”保存时调用的是 `/maintenance-task/save`（新建/兼容保存接口），未走独立更新接口。
2. 前端保存逻辑缺少 `try/catch`，接口报错时用户侧表现为“点击保存没反应”。
3. 后端缺少 `/maintenance-task/update` 显式更新接口，无法与前端编辑场景形成清晰接口约定。

## 改动文件
1. `frontend/src/views/MaintenanceTaskDetail.vue`
   - 编辑保存改为调用 `/maintenance-task/update`
   - 增加成功/失败提示处理
   - 规范 `taskNo` 判空与去空格
2. `backend/src/main/java/com/distributionbox/controller/InspectionTaskController.java`
   - 新增 `POST /maintenance-task/update`（兼容 `/inspection-task/update`）
   - 校验 `id/taskNo`
   - 使用 `updateById` 持久化维保工单字段
3. `docs/demands/v1.55.md`
4. `docs/done/v1.55.md`

## 验证结果
1. frontend：`npm run build` 通过。
2. backend：`mvn -DskipTests compile` 通过。
3. 代码链路核对：
   - 详情页保存按钮调用更新接口 `/maintenance-task/update`
   - 后端更新接口接收并更新 `taskNo/maintenanceType/maintenanceLocation/inspectionTime/maintenanceContent/maintenanceSummary`

## 风险与回滚
1. 风险：
   - 更新接口使用 `updateById`，若前端未来传入空值，会按空值覆盖对应字段（当前编辑表单为完整字段，风险可控）。
2. 回滚：
   - 回滚本次提交即可恢复旧逻辑（前端保存回到 `/save`，后端移除 `/update`）。
