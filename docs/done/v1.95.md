# v1.95 完成记录

## 做了什么
1. 新增数据库迁移 `V95__replace_maintenance_with_box_scoped_month_quarter_year_tables.sql`：
   - 删除旧 maintenance 表：`sys_maintenance_task`、`sys_maintenance_disease`、`sys_maintenance_compare_image`、`sys_maintenance_process_image`。
   - 新增三张维保宽表：
     - `sys_box_maintenance_monthly`
     - `sys_box_maintenance_quarterly`
     - `sys_box_maintenance_yearly`
   - 各表包含：`box_id`、`supervise_user`、`maintenance_user`、检查项结果/是否正常/备注字段、创建/更新时间与索引。
2. 新增后端接口 `BoxMaintenanceController`：
   - `GET /box-maintenance/{type}/page`
   - `GET /box-maintenance/{type}/{id}`
   - `POST /box-maintenance/{type}/save`
   - `DELETE /box-maintenance/{type}/{id}`
   - 支持 `type=monthly|quarterly|yearly`，并校验 `boxId`。
3. 前端 `BoxDetail.vue` 新增“维保记录”Tab：
   - 下拉选择月检/季检/年检。
   - 列表展示当前配电箱 `box.id` 下对应类型记录。
   - 新增弹窗包含：盯控人员、维保人员 + 固定检查项表格（检查内容/检查标准写死，结果/是否正常/备注可填）。
   - 保存时自动带当前 `box.id`。
4. 下线旧 maintenance 入口：
   - 移除前端路由 `/maintenance`、`/maintenance-task/:id`。
   - 移除侧边栏“维保记录”旧入口。

## 为什么这么改
- 满足“维保仅在配电箱详情页创建并强绑定 `box.id`”的业务规则。
- 采用月/季/年三张宽表，符合“检查项作为字段、一次维保一条数据”的数据模型。
