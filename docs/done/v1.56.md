# v1.56 完成记录

## 根因
线上实际运行的后端实例未包含新加的 `POST /maintenance-task/update`，导致前端调用更新接口时返回 405。

## 改动
- 文件：`frontend/src/views/MaintenanceTaskDetail.vue`
- `saveTask` 调整为：
  1. 优先调用 `/maintenance-task/update`
  2. 若返回 405，自动回退调用 `/maintenance-task/save`
  3. 完整补齐成功/失败提示与回显刷新

## 效果
- 兼容新旧后端同时可保存。
- 用户侧不再出现“点击无反应”。

## 回滚
- 回滚本次提交即可恢复只调用 `/maintenance-task/update` 的逻辑。
