# v1.23 完成记录（管理员全权限 + users/roles/menus 限权）

## 改动点

### 1) 后端
- 启用并收敛拦截器范围：
  - `src/main/java/com/xjy/shujia/config/InterceptorConfig.java`
  - 从“全量放行”改为仅放行登录/注册与文档接口，其余请求统一进入 JWT 校验。
- 在 JWT 拦截器中增加管理员专属资源判断：
  - `src/main/java/com/xjy/shujia/config/interceptor/JwtInterceptor.java`
  - 规则：`/user(s)/**`、`/role(s)/**`、`/menu(s)/**` 仅 `ROLE_ADMIN` 可访问。
  - 非管理员访问时抛出 403 业务异常。
- 补充 403 业务码常量：
  - `src/main/java/com/xjy/shujia/common/Constants.java`
- 全局异常处理返回正确 HTTP 状态：
  - `src/main/java/com/xjy/shujia/exception/GlobalExceptionHandler.java`
  - 401 -> HTTP 401，403 -> HTTP 403，避免误返回 500。

### 2) 前端
- 菜单层隐藏管理员资源入口：
  - `frontend/src/layouts/MainLayout.vue`
  - 为菜单增加 users/roles/menus 项，同时对普通用户过滤这些路径。
- 路由守卫二次拦截：
  - `frontend/src/router/index.ts`
  - 普通用户直输 `/users`、`/roles`、`/menus` 时统一跳转 `/box`。

## 为什么这么做
- 后端拦截保证“真实安全边界”，防止仅靠前端隐藏导致的越权。
- 前端守卫改善体验，避免普通用户进入无权限页面后再报错。
- 通过显式 403 响应，清晰区分“无权限”与“系统异常”。

## 验证步骤
1. 后端编译：`cd backend && mvn -DskipTests compile`
2. 前端构建：`cd frontend && npm run build`
3. Admin 验证：
   - 登录后可见并访问 `/users`、`/roles`、`/menus`。
4. User 验证：
   - 登录后菜单不显示 users/roles/menus。
   - 手输上述路由会被前端守卫跳回 `/box`。
   - 直接请求后端相关接口返回 403（而非 500）。
