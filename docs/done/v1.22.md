# v1.22 完成记录（统一图片上传接口 + 全站图片字段上传改造）

## 做了什么
1. 后端新增统一图片上传接口 `POST /files/image/upload`，并将旧接口 `/files/upload` 兼容到同一逻辑。
2. 上传返回标准化数据（`url/filename/size/contentType/uuid/previewUrl`）。
3. 增加基础安全校验：扩展名白名单、MIME 校验、大小限制（默认 10MB，可配置）。
4. 前端将所有图片字段从“手输 URL”改为 `ImageUpload` 本地上传组件，上传后自动回填并预览。
5. 新增数据库迁移 `V27`，统一扩展图片 URL 字段长度到 `VARCHAR(1000)`，兼容更长链接。

## 为什么这样改
- 上传入口统一后，业务侧无需维护多个上传规则。
- 录入体验由“复制 URL”变为“直接上传”，减少人为错误。
- 历史 URL 数据不变，保证平滑升级。
- URL 扩容避免后续 CDN/签名链接超长导致截断问题。

## 验证
### 构建验证
1. `cd backend && mvn -DskipTests compile`
2. `cd frontend && npm run build`

### 端到端验证（示例）
1. 打开“配电箱”新增弹窗，在“图片1”点击上传本地 png/jpg。
2. 上传成功后表单自动显示图片预览，字段自动写入 URL。
3. 点击保存后重新编辑该记录，图片 URL 与预览可正常回显。
