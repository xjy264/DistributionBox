# v1.26 完成记录

## 本次改动
1. 修复 `BoxDetail` 基础信息“偶发不显示/显示异常”问题，保证以下字段稳定渲染：`id`、`boxId`、`station`、`area`、`boxAddress`、`size`。
2. 保持详情页既有标签页能力不变：元器件、巡检、抢修数据与操作仍按原入口加载与保存。
3. 增加缺失/延迟数据兜底：页面展示统一安全降级为 `-`，避免空字符串、`null`、`undefined` 直接暴露。

## 修改文件
- `frontend/src/views/BoxDetail.vue`
- `docs/done/v1.26.md`

## 根因分析
- 详情页原先把“基础信息 + 元器件 + 巡检 + 抢修”放在同一个 `Promise.all` 中加载。
- 任一标签页接口失败（或短时异常）都会导致整个 `load()` 失败，进而连基础信息也不赋值，造成基础信息“有时不显示”。

## 具体修复
1. 将详情页加载改为 `Promise.allSettled`，每类数据独立处理，互不阻塞。
2. 新增响应归一化逻辑，兼容“原始数据结构”和“`Result.data` 包装结构”。
3. 为表头/描述信息增加统一 `toDisplay()` 输出函数，缺失值显示 `-`。
4. 增加路由参数监听（`watch(route.params.id)`）与请求令牌（`currentLoadToken`），避免快速切换详情时旧请求回写覆盖新数据。

## 验证
1. 前端构建验证通过：
   - 命令：`npm run build`
   - 结果：`vite build` 成功（无编译错误）
2. 代码层确认：

   - 标签页接口失败时，仅对应列表回退为空数组，不再影响基础信息渲染。
